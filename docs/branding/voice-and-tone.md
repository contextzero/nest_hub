# Voice and Tone — NEST

## Voice Principles

| Principle | Description |
|-----------|-------------|
| **Confident** | We know what we built; we explain it clearly |
| **Clear** | Technical but accessible |
| **Direct** | Get to the point |
| **Helpful** | Anticipate questions |

---

## Tone Spectrum

```
Casual ←————————————→ Professional ←————————————→ Technical

Quickstart: "5 minutes to your first agent"
     ↓
Install: "Follow these steps to configure..."
     ↓
API: "GET /api/agents — Returns all agents..."
```

---

## Writing Rules

### Do

- Use active voice
- Use second person ("You")
- Explain technical terms
- Provide code examples
- Include error guidance

### Don't

- Use filler ("simply", "just")
- Assume knowledge
- Write long paragraphs
- Use marketing hype
- Leave readers stuck

---

## Error Messages

**Format:**

> **Error:** [What happened]
> 
> **Cause:** [Why]
> 
> **Fix:** [How to solve]
