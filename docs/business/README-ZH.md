## 执行摘要

NEST 改变组织部署 AI 智能体的方式——从孤立工具转向在您的基础设施上协同运作、可在手机上查看、受您策略治理的协调劳动力。

本文档为评估与采用 NEST 提供战略背景。

## 为何选择 NEST？——第一性原理分析

### 问题（现状）

| 相关方 | 痛点 | 根因 |
|-------------|------------|------------|
| **创始人/管理员** | 无法洞察 AI 活动 | 工具各自为政 |
| **IT 负责人** | 安全与合规缺口 | 缺少审计轨迹 |
| **团队负责人** | 无法监督分布式智能体 | 无移动端管控 |
| **员工** | 会话间上下文丢失 | 记忆无法持久 |
| **NGO / 政府** | 捐赠者/公民报告无审计轨迹 | 手工流程，无合规可见性 |

### 方案（NEST 价值）

| 相关方 | 主要收益 | 为何重要 |
|-------------|-----------------|----------------|
| **创始人/管理员** | 全面掌控、无供应商锁定 | 您的基础设施、您的密钥、您的数据 |
| **员工** | 手机/平板语音办公 | 真正的移动优先运营 |
| **企业** | 所有智能体的单一枢纽 | 消除工具碎片化 |
| **NGO / 政府** | 合规就绪审计 + 移动外勤 | 捐赠者报告、公民服务、轮岗人员上下文 |

## 四大支柱

### 1. 企业级 Hub

一款应用替代 Slack + Notion + Trello + WhatsApp。项目 → 员工 → 会话。

- 集中管理所有 AI 智能体
- 按项目组织、基于角色的访问
- 完整可见性与审计轨迹
- Self-hosted：您的基础设施、您的数据

### 2. 员工移动端

任意手机上的 PWA。在公交车上审批。无需笔记本。

- 通过 Progressive Web App 完整移动操作
- 实时审批与会话控制
- 语音界面（ElevenLabs）加快输入
- 4G 或 Wi-Fi 均可——真正的远程操作

### 3. Memory Bank (Souls)

Hub 学习每位员工。无需重复解释上下文。智能持续积累。

- 员工档案保存偏好与工作模式
- 项目记忆保留决策、术语表与上下文
- Colony memory 发现最优工作流
- 知识不离开您的基础设施

### 4. Agent Swarm

多个专业 AI 智能体并行协作。

- Claude Code, Codex, Cursor, Gemini, OpenCode, KiloCode
- ZeroClaw：无界面自动化与自纠错
- OpenClaw：可视化工作流编排
- Darwin Agents：随组织演进的 AI 工作者

## 竞争差异化

| 竞品 | 其弱点 | NEST 优势 |
|------------|----------------|-----------------|
| **Zapier** | 基于规则的自动化 | AI 原生，学习并演进 |
| **Salesforce** | 侧重销售 | 每位员工，不限销售 |
| **UiPath** | 基于规则的 RPA | AI 原生，非规则驱动 |
| **GitHub Copilot** | 仅编码 | 完整劳动力，不限代码 |
| **Agentforce** | 仅 SaaS | Self-hosted 选项，完全掌控 |
