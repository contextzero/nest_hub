# AGENTS.md

This file provides guidance on how to work with the `nest_hub` repository.

## Project Overview

**NEST** is a self-hosted AI workforce hub by **Context Zero (CTX0)** that
consolidates fragmented AI tooling into a single, governed plane: organizational
memory, multi-agent orchestration, 700+ models via BYOK, and full audit trails —
all on infrastructure the customer owns.

**This repository is the self-hosted _distribution_ layer**, not the product
source. It ships the Docker Compose stack, reverse-proxy configuration, install
scripts, public assets, and the documentation set. The application itself is
delivered as **prebuilt artifacts**:

| Component | Technology | Distribution |
|-----------|-----------|--------------|
| **Server** | Rust · Axum · Socket.IO · SSE | Docker image `matiasbaglieri/nest-server` |
| **Web App / PWA** | React · Vite · TanStack · Tailwind · xterm.js | Docker image `matiasbaglieri/nest-web` |
| **CLI** | TypeScript · Bun · Ink | npm `@contextzero/nest` (binary: `annie`) |
| **Database** | PostgreSQL 16 | Docker (internal-only, not exposed) |
| **Proxy** | nginx Alpine | Docker |

> **There is no Rust/React/CLI source in this repo.** Do **not** add `cargo
> build`, `pnpm build`, or "build from source" instructions here — they will not
> work. The supported flow is `./setup.sh` → `docker compose`.

## General Guidelines

- This repo is **deployment + docs**. Most changes touch `docker-compose.yml`,
  `nginx/`, `apache/`, `setup.sh`, `public/`, or `docs/` and the root `.md` set.
- Keep documentation **concise and accurate to the shipped stack**. Prefer
  linking into `docs/` over duplicating long content in the README.
- The **README is the canonical product description**; keep module names and
  claims in sync across `README*.md`, `QUICKSTART*.md`, `ROADMAP.md`, and
  `docs/`.
- Treat secrets (`POSTGRES_PASSWORD`, `CLI_API_TOKEN`, `ENCRYPTION_KEY`) as
  sensitive. `.env` is generated by `setup.sh`; never commit real secrets.
- Use mermaid diagrams in `.md` files when a visual helps.

## Essential Commands

All commands run from the repo root.

```bash
./setup.sh                                   # First-time setup (auto-generates secrets, pulls images, waits for health)
docker compose up -d                         # Start the stack
docker compose down                          # Stop the stack
docker compose logs -f                       # Stream logs
docker compose ps                            # Check status
docker compose pull && docker compose up -d  # Update to the latest published images
```

The hub is served by nginx on `WEB_PORT` (default `80`). PostgreSQL is
internal-only (no published port); emergency admin is
`docker exec -it nest-db psql -U postgres -d nest`.

## Documentation Conventions

- **English-only docs:** the documentation set is English-only. Earlier
  `-ES`/`-ZH`/`-DE`/`-PT`/`-FR` translations were removed to keep the repo lean;
  do not reintroduce per-language files without discussing it first.
- **Where things live:**
  - First-time Docker setup → `QUICKSTART.md` → `docs/INSTALL.md`
  - Public URL + TLS / production → `docs/DEVOPS.md`
  - CLI (`@contextzero/nest`) + LLM/BYOK config → `docs/CLI-BUSINESS.md`,
    `docs/enterprise/annie-cli-mcp-enterprise.md`
  - Enterprise positioning → `docs/enterprise/README.md`
  - Shipped vs planned → `RELEASES.md` / `ROADMAP.md`
- **Version notifications** are JSON files under `public/updates/` (see commit
  history: `chore: update notification JSONs`).
