# Phases — NEST Product Evolution

The roadmap from a single tool to a complete workforce automation platform.

---

## Phase 0 — Discovery (Validation)

**Focus:** Validate core value proposition

- 1–2 critical flows: "mobile approvals" + "remote session control"
- Metric: time-to-first-task shipped from a phone

**What's Built:**
- Basic session management
- Mobile PWA with approval flow
- Single agent mode (Claude Code)

**Status:** ✅ Complete

---

## Phase 1 — Foundation (Operational Hub)

**Focus:** Make it usable for a team

- Admin config (LLMs + voice)
- Core Users/Projects
- Multi-session + approvals
- Minimum telemetry (tokens, latency, audit)

**What's Built:**
- Multi-user support
- Project organization
- Session visibility dashboard
- Basic audit logging

**Status:** ✅ Complete

---

## Phase 2 — Productivity (Cost + Quality)

**Focus:** Make it efficient and cost-effective

- Per-project indexing and "context packs"
- Memory bank per project/user
- Assisted prompting (swarm to improve prompts)
- Templates for recurring tasks

**What's Built:**
- Project indexing
- Memory system
- Prompt templates
- Multi-agent support (Claude, Codex, Cursor, OpenCode, Gemini, KiloCode)

**Status:** ✅ In Progress

---

## Phase 3 — Spec-Driven Development (SSD)

**Focus:** Structured task execution

- Specs as a standard input
- Planner → subtasks
- Auto mode vs approval mode
- Tracking of progress and artifacts

**What's Built:**
- Task graph planning
- Auto/Approval mode toggle
- Progress tracking
- Artifact management

**Status:** 🚧 Building

---

## Phase 4 — Automation at Scale (OpenClaw / ZeroClaw)

**Focus:** Beyond coding — full automation

- Machine automation (browser/terminal)
- End-to-end playbooks
- Strong security policies + advanced audit

**What's Built:**
- OpenClaw (project orchestration)
- ZeroClaw (headless automation with self-correction)
- Playbook system

**Status:** 🚧 Building

---

## Phase 5 — Media Workflows

**Focus:** Beyond text — video and images

- Video generation integration (WAN/HeyGen-like)
- Image generation integration (NanoBanana-like)
- Queues, assets, approvals, and publishing

**What's Built:**
- Architecture planned
- Integration points defined

**Status:** 📋 Roadmap
